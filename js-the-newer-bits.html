JS - The newer bits

Here we are going to look at some of the more recent changes in javaScript. These things are more about syntactical changes, easier ways of doing things that we could already do it javaScript. It's important to know these things as you will likely come across them along your journey.

Default Params

Default params (default parameters) are often encountered in javaScript. Lets say we have a function, sometimes the params we pass in are optional. If these params are not filled; we would like to give them a default value.

function rollDie(numSides) {
return Math.floor(Math.random() * numSides) + 1;
}

Here our rollDie function that we've seen before. Tis function takes in a number of sides and returns a random number multiplied by the number of sides the die has, plus 1.

If we call roleDie in the console and give it a number, we will get a number returned to us within the parameters.

// output

rollDie(5)
// 4

We asked the function to give us a number between 1 and 5, we got 4. But what happens if the user doesn't pass in the params?

// output

rollDie()
// NaN

We get NaN returned to us, because this parameter is undefined.

The old way of getting around this problem is like this.

The Old Way

function rollDie(numSides) {
  if (numSides === undefined) {
  numSides = 6;
  }
  return Math.floor(Math.random() * numSides) + 1;
}

Now if we call rollDie in our console we get this.

rollDie( )
2
rollDie( )
5
rollDie( )
3
rollDie( )
4

A random die role between 1 and 6 without passing in the parameter, because we've checked within the function if the numSides is undefined and passed in a default param. This way of doing things can be annoying, especially if we have more than one parameter passed in.

The New Way

function multiply(a, b = 1) {
  return a * b;
}

In this example, a has no default value, a will be whatever you pass in as the default argument. b on the other hand, does have a default value, if its not defined, it will default to 1. We can rewrite our 'old way' function to use the newer way.

function rollDie(numSides = 6) {
  return Math.floor(Math.random() * numSides) + 1;
}

We loose a whole if statement and set the default params in a much more concise, readable way.

Multiple Default Params

We can write multiple default params, but the important thing is the order they are written in.

function greet(msg, person) {
  console.log(`${msg}, ${person}`);
}

Here we have a function called greet that takes in two params, a message and a person. Whatever message and person we pass in, will be returned to us in the console.

function greet(msg, person) {
  console.log(`${msg}, ${person}`);
}

Lets try and pass in a default param and see what happens.

function greet(msg = "Hey there", person) {
  console.log(`${msg}, ${person}`);
}

// output

greet("Tom");
// Tom undefined

javascript doesn't know that we are passing in a person here. It's simply going from the order. 'Tom' was the first thing we passed in, so its going to be applied to the first param it finds. In this case, person is left undefined. You should always have your default params come after the first param to prevent problems like this occuring.
