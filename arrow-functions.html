Arrow Functions

An arrow function is a syntactically compact alternative to a regular function.

Some simple arrow functions.

const square = (x) => {

return x * x
}
const sum = (x, y) => {

return x + y
}

As you can see we don't need the compelete function expression that we have previously used, though arrow functions work exactly the same. The required features of an arrow function are () if no argument is passed in, and => the arrow syntax.

Lets write a simple arrow function called greet that accepts a single string argument, representing a person's name.

const greet = (name) => {
  return `Hey ${name}!`
}

// output

greet('Tom')
'Hey Tom!'

Using a template literal we take the information passed into the name argument through the arrow function, and return a string when it's called.

Implicit Return

Arrow functions are very compact compared to a typical function expression, but there is a way to make them even more compact. We can take advantage of implicit return. This can only be done with arrow functions. It means that we can leave off the return keyword in certain situations.

Below is a normal function, we will adapt it to make it an implicit return function.

const rollDie = () => {

return Math.floor(Math.random() * 6) + 1
}
const rollDie = () => (

Math.floor(Math.random() * 6) + 1
)

We have made some slight changes to the above function. We have lost the return keyword, and substituted our curly braces for brackets. This is the correct syntax for an implicit return.

const rollDie = () => Math.floor(Math.random() * 6) + 1

For very short functions we can even get rid of those parenthesis and just have our function displayed on one line. These functions are not necassary to use, but they can make your javaScript code more consise.

Arrow functions and the 'this' keyword

Arrow function behave very differently in terms of the value for the keyword 'this' comapred to a traditional function.

const person = {
  firstName: 'Viggo',
  lastName: 'Mortensen',
  fullName: function () {
  `${this.firstName} ${this.lastName}`;
  },
};

Above we have an object called person with two values inside, a 'firstName' and a 'lastName'. We then run a function on full name to combine both first and last names with string templae literals and the 'this' keyword. See the output below.

// output

// 'Viggo Mortensen'

In this case, the 'this' keyword applies to whatever is to the right of it, hence why we get the combined 'fullName' returned.

Lets see what happens if we use an arrow function with the same logic.

const person = {
  firstName: 'Viggo',
  lastName: 'Mortensen',
  fullName: () => {
  `${this.firstName} ${this.lastName}`;
  },
};

// output

// 'undefined undefined'

Lets take a look at what is happening here. Inside of an arrow function, the keyword 'this', is just going to refer to the scope that it was created in. That means that in this case, the keyword 'this' refers to the window object. When we use the keyword 'this' in a traditional function, it has nothing to do with the scope of where it was created, it has to do with how the function is executed. This is by design, because at times it can be frustrating having to worry about the execution context and how that impacts the keyword this.

const person = {
  firstName: 'Viggo',
  lastName: 'Mortensen',
  fullName: () => {
  `${this.firstName} ${this.lastName}`;
  },
shoutName: function() {
 setTimeout(function () {
   console.log(this.fullName ()) {
   }, 3000)
  }
}

Above we have added a function that will shout the full name after a setTimeout of threee seconds.

// output

// 'undefined'
Uncaught TypeError: This.fullName is not a Function

You can see in our output above that the original call of shoutName returned undefined, which we would have expected. Then after we wait three seconds for the setTimeout we get an error message, this is because setTimeout( ) is part of the window object.

The main point here is that the keyword 'this' reacts differently when it is used in an arrow function, compared to when its used in a regular function.
